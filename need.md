# 抖音内容收集器 — 需求文档

## 1. 产品定位

一款安卓 App，自动归档所关注博主在抖音上公开发布的作品（图文+视频文案），形成个人离线内容库。兼具**信息流浏览**和**博主归档存档**两种使用模式。

## 2. 目标用户

- 关注多位抖音博主，希望系统性地回顾其历史内容
- 不想被平台算法和信息流淹没，希望按自己的节奏查阅
- 需要离线保存，随时翻看

## 3. 核心功能

### 3.1 博主管理
- **添加博主**：支持输入抖音号（`@xxx`）或粘贴博主主页链接
- **移除博主**：一键移除，同时删除该博主的所有归档内容
- **博主列表**：显示头像、昵称、抖音号、粉丝数

### 3.2 内容自动归档
- **自动同步**：下拉刷新触发，逐个拉取所有博主的最新作品
- **全量保存**：
  - 作品文案（description）
  - 图文作品的图片（多图）
  - 视频封面图
  - 互动数据（点赞、评论、播放、分享数）
  - 发布时间、分享链接
- **增量更新**：已存在的作品跳过，只保存新作品
- **离线存储**：所有数据存入手机本地 SQLite 数据库，断网可查

### 3.3 内容浏览

#### 信息流模式
- 所有博主的内容按发布时间倒序排列
- 卡片式展示（封面图 + 文案摘要 + 互动数据 + 发布时间）
- 标记作品类型（图文/视频）

#### 博主归档模式
- 按博主维度的历史作品列表
- 左侧缩略图 + 右侧文案的紧凑布局
- 显示该博主的总作品数

#### 作品详情
- 完整文案
- 图文作品支持左右滑动浏览多图（带页码指示器）
- 视频作品显示封面 + 播放按钮占位
- 互动数据展示（点赞、评论、分享、播放数）
- 作品元信息（作品ID、类型、发布时间）

### 3.4 状态反馈
- 同步时显示进度提示
- 操作成功/失败通过 SnackBar 提示
- 空状态引导（无内容时提示添加博主）

## 4. 技术方案

| 模块 | 技术选型 |
|------|---------|
| 框架 | Flutter 3.22.x (Dart) |
| 本地存储 | SQLite (sqflite) |
| 网络请求 | Dio + CookieJar |
| 图片缓存 | cached_network_image |
| 状态管理 | Provider |
| 最低 Android 版本 | API 21 (Android 5.0) |

## 5. 数据来源

通过抖音网页版公开 API 获取，仅采集**公开发布**的内容：
- 用户信息 API：`/aweme/v1/web/user/profile/other/`
- 作品列表 API：`/aweme/v1/web/aweme/post/`
- 搜索 API：`/aweme/v1/web/discover/search/`

不涉及：模拟登录、破解客户端、获取私密数据。

## 6. 数据模型

### 博主 (Creator)
| 字段 | 说明 |
|------|------|
| secUid | 抖音唯一标识 |
| uniqueId | 抖音号 (@xxx) |
| nickname | 昵称 |
| avatarUrl | 头像链接 |
| followerCount | 粉丝数 |
| addedAt | 添加时间 |
| lastFetchedAt | 最后同步时间 |

### 作品 (Post)
| 字段 | 说明 |
|------|------|
| awemeId | 作品ID |
| creatorId | 所属博主 |
| description | 文案 |
| imageUrls | 图片链接列表 |
| videoCoverUrl | 视频封面 |
| publishTime | 发布时间 |
| viewCount / likeCount / commentCount / shareCount | 互动数据 |

## 7. 界面结构

```
底部导航
├── 信息流 (首页)
│   └── 作品卡片列表 (时间倒序)
│       └── 点击 → 作品详情页
└── 博主
    ├── 博主列表
    │   ├── 点击 → 博主详情 (归档)
    │   └── FAB → 添加博主
    ├── 博主详情页
    │   ├── 博主信息头 + 作品数
    │   └── 作品列表
    └── 添加博主页
        └── 输入框 + 添加按钮
```

## 8. 编译与发布

- 源码托管：GitHub (zzk-vm/douyin-collector)
- 自动编译：GitHub Actions (push 到 master 自动触发)
- 分发方式：GitHub Releases (每次编译自动生成下载链接)
- 安装方式：用户下载 APK → 手机侧载安装

## 9. 边界说明

### 能做
- ✅ 追踪抖音公开博主的公开发布内容
- ✅ 离线归档，随时查阅
- ✅ 个人学习研究用途

### 不做
- ❌ 不追踪私密账号/好友圈内容
- ❌ 不爬取微信朋友圈、QQ空间
- ❌ 不模拟登录他人账号
- ❌ 不修改/删除抖音上的数据
- ❌ 不对外分发收集的内容
